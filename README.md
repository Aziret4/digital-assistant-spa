# Цифровой помощник для малого бизнеса

SPA-приложение для предпринимателя малого бизнеса (на примере ателье).
Позволяет вести клиентскую базу, создавать заявки, переводить их в заказы,
отслеживать статусы и использовать цифрового помощника для подготовки
ответов клиентам.

Выпускная квалификационная работа.

## Стек технологий

**Frontend:**
- React 19 + Vite
- React Router (маршрутизация)
- Axios (HTTP-клиент)
- CSS (без UI-фреймворков)

**Backend:**
- Node.js + Express
- PostgreSQL (драйвер `pg`, без ORM)
- JWT для авторизации (`jsonwebtoken`)
- Хэширование паролей (`bcryptjs`)

**Архитектура:** клиент-серверная, REST API.

## Установка и запуск

### 1. Клонировать проект

```bash
git clone <url-репозитория>
cd digital-assistant-spa
```

### 2. Настроить базу данных

1. Открыть **pgAdmin** или psql
2. Создать базу данных: `CREATE DATABASE digital_assistant;`
3. Подключиться к этой базе и выполнить SQL-скрипт:
   ```sql
   -- скопировать содержимое server/src/db/schema.sql и выполнить
   ```

### 3. Настроить backend

```bash
cd server
cp .env.example .env
```

Открыть `.env` и заполнить реальными значениями:

```
PORT=5000

DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=ваш_пароль_от_postgres
DB_NAME=digital_assistant

JWT_SECRET=любая_длинная_случайная_строка
JWT_EXPIRES_IN=7d
```

Установить зависимости и запустить:

```bash
npm install
npm run dev
```

Сервер запустится на `http://localhost:5000`.

Проверка: `http://localhost:5000/api/health` должен вернуть `{"status":"ok"}`.

### 4. Настроить frontend

В новом терминале:

```bash
cd client
npm install
npm run dev
```

Frontend откроется на `http://localhost:5173`.

## Учётная запись по умолчанию

При первом запуске нужно зарегистрироваться через страницу **Регистрация**
либо использовать Postman:

```
POST http://localhost:5000/api/auth/register
Content-Type: application/json

{
  "name": "Администратор",
  "email": "admin@test.com",
  "password": "123456"
}
```

После регистрации данные для входа:
- **Email:** `admin@test.com`
- **Пароль:** `123456`

## API endpoints

Все запросы, кроме `/api/auth/*`, требуют заголовок:
```
Authorization: Bearer <JWT-токен>
```

### Авторизация

| Метод | Endpoint | Описание |
|-------|----------|----------|
| POST | `/api/auth/register` | Регистрация (возвращает JWT) |
| POST | `/api/auth/login` | Вход (возвращает JWT) |

### Клиенты

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/api/clients` | Список клиентов |
| GET | `/api/clients/:id` | Один клиент |
| POST | `/api/clients` | Создать клиента |
| PUT | `/api/clients/:id` | Обновить клиента |
| DELETE | `/api/clients/:id` | Удалить клиента |

### Заявки

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/api/requests` | Список заявок (с именем клиента через JOIN) |
| GET | `/api/requests/:id` | Одна заявка |
| POST | `/api/requests` | Создать заявку |
| PUT | `/api/requests/:id` | Обновить заявку (включая смену статуса) |
| DELETE | `/api/requests/:id` | Удалить заявку |
| POST | `/api/requests/:id/convert` | Перевести заявку в заказ (SQL-транзакция) |

### Заказы

| Метод | Endpoint | Описание |
|-------|----------|----------|
| GET | `/api/orders` | Список заказов |
| GET | `/api/orders/:id` | Один заказ |
| POST | `/api/orders` | Создать заказ |
| PUT | `/api/orders/:id` | Обновить заказ (включая смену статуса) |
| DELETE | `/api/orders/:id` | Удалить заказ |

### Цифровой помощник

| Метод | Endpoint | Описание |
|-------|----------|----------|
| POST | `/api/assistant/ask` | Получить подсказку ответа на вопрос клиента |

На первом этапе помощник работает на основе правил (ключевых слов).
Архитектура позволяет заменить логику на вызов LLM (OpenAI / локальная модель)
без изменения других слоёв приложения.

## Основные возможности

### Авторизация
- Регистрация и вход по email и паролю
- Хэширование паролей через `bcryptjs`
- JWT-токены (срок жизни 7 дней)
- Защищённые маршруты на frontend (компонент `ProtectedRoute`)
- Автоматическая подстановка токена в запросы через `axios.interceptors`

### Клиенты
- Полный CRUD
- Поля: ФИО, телефон, email, источник обращения (WhatsApp / Instagram /
  Telegram / телефон / сайт / другое), заметки

### Заявки
- Полный CRUD
- Статусы: новая, в обработке, подтверждена, отклонена, переведена в заказ
- Цветовая индикация статусов
- Связь с клиентом через foreign key
- В списке отображается имя клиента (через SQL JOIN)

### Заказы
- Полный CRUD
- Статусы: принят, в работе, готов, выдан, отменён
- Поля: название услуги, сумма, срок выполнения, комментарий
- Форматирование суммы и дат в русском локале

### Перевод заявки в заказ
- Отдельный эндпоинт `POST /api/requests/:id/convert`
- Реализован через **SQL-транзакцию** (BEGIN / COMMIT / ROLLBACK)
- Атомарно создаёт заказ и обновляет статус заявки
- При ошибке — откат всех изменений

### Цифровой помощник
- Принимает вопрос клиента, возвращает готовый текст ответа
- Mock-логика на основе ключевых слов
- Кнопка копирования ответа в буфер обмена
- Примеры вопросов в виде кликабельных подсказок

### Dashboard
- Агрегированная статистика: количество клиентов, активных заявок, заказов в работе
- Параллельная загрузка данных через `Promise.all`
